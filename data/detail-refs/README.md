# 안경 디테일 실사진 풀 (6매장 공유)

gpt-image(gti)는 안경 부품 접사(코받침·힌지·렌즈 두께 등)를 왜곡해서 못 그린다.
매장 전경을 실제 사진으로 서빙하듯(메모리 v2.3), **안경 디테일 컷도 실제 사진을 직접 서빙**한다.
이 디테일들은 매장과 무관한 일반 컷이라 매장별로 모으지 않고 **여기 공유 풀** 한 곳에 둔다.

## 어디에 뭘 넣나 (주제별 하위폴더)

| 폴더 | 넣을 사진 | 쓰이는 글 |
|------|----------|----------|
| `nose-pad/` | 코받침·코패드 종류(실리콘/금속/일체형), 콧등 자국, 패드 클로즈업 | 코받침 자국·피팅·흘러내림 |
| `lens/` | 렌즈 두께·굴절률 옆모습, 코팅 반사, 변색/편광 렌즈 | 고굴절·압축·블루라이트·변색렌즈 |
| `frame/` | 안경테 소재(티타늄/뿔테/메탈/울템), 힌지·나사·다리 등 부품 | 안경테·수리·소재 |
| `contacts/` | 콘택트렌즈 알·케이스·착용 디테일 | 콘택트렌즈 |
| `general/` | 위에 안 맞는 일반 안경 클로즈업(폴백) | 매칭 안 될 때 |

## 사진 요건
- **실제 사진**(직접 촬영 또는 사용권 확보된 것). 임의 웹 크롤링 금지.
- 글자·로고가 안 보이는 깔끔한 컷이 좋음(있어도 워싱 단계에서 처리되지만 없는 게 안전).
- 가로형(4:3 근처) 권장. 파일명은 자유(한글 OK). 같은 사진을 여러 매장이 써도 워싱(imageWash)으로 매번 다른 해시 → 네이버 중복 회피.
- 폴더당 3~10장 정도면 충분(매번 랜덤 1장 워싱 서빙).

## 코드 연결
사진을 넣으면 [SCENE:detail] 슬롯에서 글 주제에 맞는 폴더의 실사진을 골라 서빙한다.
주제 폴더가 **비어 있으면** 무관한 general 공통컷으로 때우지 않고 그 detail 컷을 **드롭**한다
(`listDetailCategoryPhotos` strict + prompts/route 드롭). 즉 폴더에 사진이 있어야 그 주제 detail이 나온다.

## ⚠ AI 임시본 (`ai_*.png`)
실사진이 부족한 폴더(lens·contacts·nose-pad)를 임시로 채우려고 **AI로 생성한 placeholder**다
(2026-06-18, gti, 왜곡 검수 통과분만). 파일명 `ai_` 프리픽스로 구분.
**사장님 실물 촬영본이 들어오면 이 `ai_*.png`들을 지우고 교체할 것** — 진짜 사진이 항상 우선.
