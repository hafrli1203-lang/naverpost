# 매장별 실제 참조사진 (shop-refs)

매장 장면 이미지를 **실제 매장 모습**에 맞추기 위한 폴더 (v2.0 — 장면 매칭).
핵심 원칙: 사진을 **참조로 활용**해 새 장면을 생성한다. 사진을 결과물에 그대로 붙여넣지(합성) 않는다.

## 어디에 넣나 (매장 = blogId 폴더)
```
data/shop-refs/
  top50jn/       ← 으뜸50안경 장림시장점
  jinysgongju/   ← 지니스안경 공주신관점
  attractiger/   ← 지니스안경 김해장유점
  leesi7007/     ← 지니스안경 대전충남대점
  kl1854/        ← 지니스안경 심곡고가사거리점
  peace9486/     ← 지니스안경 진해하나로마트점
```
각 폴더에 사진을 넣으면 된다 (파일명 자유, jpg/png/webp). 장수 제한 없음(읽기용).

## 어떤 사진을 넣나
- ✅ 매장 장면별로 골고루: **외관 / 내부 진열 / 검안실 / 피팅·조제 카운터 / 안경테 클로즈업**
- 장면 매칭이 핵심이라, 검안실·피팅 카운터 사진이 있으면 그 장면 이미지가 실제 매장과 닮게 나온다.
- 손님·직원 얼굴 크게 나온 사진은 피하는 게 좋음(인물 흉내 → 합성티).

## 장면 매칭 (v2.0 — 핵심 동작)
1. 사진을 폴더에 넣는다.
2. 각 사진을 **장면별로 분류**해 `_scene-index.json`에 기록한다(외관/내부/검안/피팅/디테일 + 매장 인테리어 묘사).
   → 사진을 보고 분류표를 채우는 작업. 새 사진 추가 시 이 인덱스 갱신 필요.
3. 이미지 생성 시 프롬프트마다 장면 태그(`[SCENE:exam]` 등)가 붙고,
   **그 장면에 맞는 매장 사진을 골라**(검안→검안실 사진, 피팅→피팅 사진) gti `--image` 참조로 첨부한다.
4. gti는 사진을 그대로 붙여넣지 않고 분위기/구도를 참고해 **새 장면을 생성** → 합성티 없이 실제 매장 톤 반영.
   - 매장 밖 장면(집·사무실·야외)·개념/비유 이미지에는 매장 사진을 붙이지 않는다.

## `_scene-index.json` 형식
```json
{
  "shops": {
    "top50jn": {
      "brand": "으뜸50안경",
      "interiorDescription": "Bright clean Korean eyewear store ... no text labels",
      "photos": {
        "5.jpg": { "scene": "exam", "desc": "검은 검안의자+HUVITZ 포롭터+옆 보조테이블" }
      }
    }
  }
}
```
- `scene` 값: `exterior` | `interior` | `exam` | `fitting` | `detail`
- `interiorDescription` : 매장 사진을 읽어 정리한 영어 묘사(프롬프트 주입용).
- 폴백: 특정 장면 사진이 없으면 interior 사진으로 대체, 그것도 없으면 묘사만으로 생성.

## profile.json (구버전 폴백)
`interiorDescription`은 이제 `_scene-index.json`에서 우선 읽는다. `profile.json`은 인덱스가 없을 때만 폴백으로 쓰인다.
`useRawReference`는 더 이상 장면 매칭 경로에서 쓰이지 않는다(장면 매칭이 기본).

## 사진이 없으면
- `interiorDescription`(또는 기본 묘사)만으로 생성. 시스템 정상 동작.
- 즉 사진은 "더 정확히" 만들기 위한 보강이지 필수는 아니다.
