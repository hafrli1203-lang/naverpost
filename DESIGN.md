# DESIGN.md — naverpost

> 이 프로젝트는 **내부 운영자용 업무 대시보드**(콘텐츠 파이프라인 도구)다. B2B 랜딩이 아니라 "입력 → 분석 → 결과 → 저장/출력" 흐름이 명확한 업무형 UI를 기준으로 본다.

## 이 프로젝트의 성격
- Human-in-the-loop 3단계 파이프라인 대시보드(키워드 → 본문 → 이미지 → export).
- 사용자가 지금 어느 단계에 있는지, 다음에 무엇을 할지가 항상 보여야 한다.

## 기술 스택 (실제 파일 근거)
- Next.js 16(App Router) + React 19, TypeScript, pnpm.
- UI: Tailwind CSS v4 + shadcn/ui(@base-ui/react), lucide-react 아이콘, next-themes(다크모드), sonner(토스트).

## 디자인 방향성 (한전ON형 + rightpeople형)
- **서비스 포털형 골격**(한전ON 방향): 명확한 상단 네비, 일관된 페이지 틀, 단계 진행 표시(stepper).
- **행동 중심 진입**(rightpeople 방향): 첫 화면에서 "다음에 할 일"(매장 선택 / 키워드 생성)이 카드형으로 보인다.
- 신뢰감 있는 절제된 업무 톤. 과한 채도·그라데이션·장식 금지. 정보 위계가 핵심.

## 화면별 중점
- 워크플로우 스테퍼: 현재 단계/완료 단계가 명확.
- 키워드 옵션: 3개 옵션을 비교·선택하기 쉽게.
- 본문 미리보기: 검증 결과(금지어/규칙 위반)를 눈에 띄게, 승인/수정 동선.
- 이미지 미리보기: 확인/재생성 동선.
- export: 복사/이미지 저장 버튼과 "네이버 임시저장" 안내를 명확히.

## 상태 처리 (필수 3종)
- 로딩(스켈레톤 권장, 스피너 지양 — AI 호출은 길어질 수 있으니 진행 표시 명확히).
- 빈 상태(다음 행동 안내 포함).
- 오류 상태(원인 + 복구 행동 — AI/네이버 API 실패 케이스 포함).

## 피해야 할 것
- 흔한 AI 그라데이션, 의미 없는 이모지, 카드 남발, 정체불명 버튼.
- 어디부터 눌러야 할지 모르는 빈 대시보드.
- 단계 흐름이 안 보이는 화면.

## 완료 기준
- 반응형, 접근성(키보드/대비/라벨, ARIA), 다크모드 동작.
- `pnpm dev`(http://localhost:3100) 실제 렌더 확인(스크린샷).
- 기존 화면/기능 회귀 없음. 한 화면씩 수정·검수.
