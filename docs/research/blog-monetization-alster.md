## 1. 네이버 검색 알고리즘 3대장 (5장) — 우리 규칙의 "왜"

| 알고리즘 | 책 설명 | 우리 시스템 대응 |
|---|---|---|
| **C-Rank** | 블로그 **전체** 신뢰도. 한 주제를 일관되게 다루면 "전문 블로그"로 인식. 자동차 블로그에 맛집 섞이면 감점 | 6매장×6카테고리 고정, 카테고리 누수 게이트 v2.4(`isCategoryAppropriateCandidate`) |
| **DIA** | **개별 글** 완성도. 글자수·이미지/영상 수·체류시간·반응(댓공) 종합. 짧은 글 불리 | 본문 ~2000자, 이미지 생성, 문단 가독성 분리(`splitKoreanParagraph`) |
| **DIA+** | **검색 의도 일치**. "간장게장 만드는 법" 검색인데 역사부터 깔면 감점. **제목=본문 정확히 일치**해야 고득점 | 제목 결정론 게이트 v27, articlePrompt의 논지(topic) 정렬·질의의도 형태소 주입 |
| **에어서치(2024.2~)** | 스마트블록 확대. 검색기록·취향 반영 개인화. 같은 키워드도 사람마다 다른 결과. **보통 3개만 노출 → 3위 안에 들어야** | `smartBlock.ts` 하위키워드 추론(라이브), articlePrompt G2 주입 |

**핵심 결론**: 우리가 이미 만든 규칙들(카테고리 일관성·제목=본문 정합·하위키워드·체류시간 문단)이 책의 3대장 + 에어서치 설명과 정확히 맞물린다. 외부 검증 성격.

## 2. 황금키워드 4조건 (4장)

1. **충분한 검색량** — 월 1,000회 이상 권장 (월 1,000 1등 ≈ 일 방문 ~20명)
2. **경쟁률** — 내 블로그 지수 대비 노출 가능한지 냉정 판단, 틈새/롱테일 우선
3. **수익화 가능성** — (우리는 브랜드 유입이 목적이라 변형 적용: "매장 방문/상담 전환 가능성")
4. **트렌드성** — 시기성 키워드

**우리가 덜 쓰는 포인트 — 유사어/오타 키워드 확장**
- 책 예: "충전기 vs 충전케이블 vs 충전선", "스탠바이미 vs 스텐바이미", "비스포크 vs 비스코프"
- 단어 하나 차이로 검색량 수배~10배. 오타 키워드도 실트래픽.
- 우리 키워드 생성기: 지역 부착 전략·하위키워드는 있으나 **표기 변형/오타/파생어 확장은 약함** → 개선 후보(미적용, 승인 대기).

대응 코드: `searchSignals.ts`(검색량), 검색광고 API 볼륨게이트, `smartBlock.ts` 자동완성 확장.

## 3. 상위노출 글쓰기 전략 (6장)

### 3-1. 상위 글 역분석 = "정답지"
- 검색 시 상위 글이 곧 네이버가 원하는 답안. 최소 5개 공통 요소 체크:
  - 제목 키워드 배치(앞/중간), 글자수(보통 **1,500자 이상**), 이미지 개수+설명 배치, 소제목(H태그) 사용, 반응(댓공스)
- 대응 코드: `competitorMorphology.ts`(상위 글 공통 명사 추출 → articlePrompt에 "질의의도 핵심 형태소" 주입), `postingAudit.ts`.

### 3-2. 반응 점수(스댓공체 = 스크랩·댓글·공감·체류) — **사람 운영자 영역**
- 발행 후 1~2시간이 황금시간(이웃 소통으로 반응 누적).
- **우리 파이프라인 범위 밖**(자동화 댓글은 책에서도 저품질 사유로 금지). → 매장 운영자 운영팁으로 전달할 것.

### 3-3. 체류시간 늘리는 글쓰기
- 3~4줄 단락, 중요부 Bold, 사진/표/인포그래픽, 스토리텔링(경험담·비교사례), 도입→본론→결론.
- 대응 코드: articlePrompt 3장(체류시간 전개 원칙, Search→…→Action), contentFormatter 문단 분리, 이미지 생성.

## 4. 누락 / 저품질 (7장) — **우리 검증 규칙과 직결**

| 저품질/누락 유발 (책) | 우리 방어 | 상태 |
|---|---|---|
| 중복 콘텐츠(자기복제·표절) | 6매장 RSS 중복 방지 | ✅ |
| 광고성 과다·단순 협찬 패턴 반복 | 금지어/의료법 필터, 홍보톤 가드 | ✅ |
| **무분별한 키워드 삽입**(같은 키워드 20번+ → **뷰누락**) | articlePrompt 키워드 출현 횟수 제한(메인/서브 고정형) | ✅ (line ~343-347) |
| 자동화 조회/댓글 조작 | 우리는 사람이 수동 임시저장 → 해당 없음 | ✅ |

**누락 3유형**: 뷰누락(뷰탭만 안 보임, 회복 가능) / 통누락(전체검색 안 보임, 패널티) / (3유형). 회복법 공통: 광고성↓ 정보성↑, 키워드 자연스럽게.

**새 인사이트**: "누락은 저품질의 전조". 키워드 스터핑 방어가 우리에게 이미 있다는 점이 중요 — 책이 그 위험을 명시적으로 확인해줌.

## 6. 액션 후보 (전부 미적용 — 승인 대기)

1. **오타/표기변형/파생 키워드 확장**을 키워드 생성기에 추가 (4장 근거). 6매장 재사용 깨지 않게 지역 비부착 원칙 유지.
2. 운영자용 "발행 후 1~2시간 반응 관리" 한 줄 가이드를 export 산출물에 동봉(선택).
3. 키워드 밀도 상한이 실제로 본문 검증(validation)에서도 강제되는지 재확인(현재는 프롬프트 지시 위주).
