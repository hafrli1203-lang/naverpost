## 운영 메모 (워크스페이스 공통)
- LLM Wiki는 프로젝트 기억 저장소다.
- Workflow는 일하는 순서다.
- Loop Pattern은 결과가 괜찮아질 때까지 반복 검수하는 장치다.
- 기능을 나눠라.
- 하나씩 구현해라.
- 직접 실행해라.
- 오류가 있으면 원인을 찾아라.
- 수정 후 다시 실행해라.
- 테스트가 통과할 때까지 완료라고 말하지 마라.
- 마지막에 무엇을 고쳤고 어떤 결과가 나왔는지 보고해라.
- 디자인도 한 화면씩 분석하고, 개선하고, 검수하고, 기록해라.

## 프로젝트 메모
- 내부 운영자용 콘텐츠 파이프라인 대시보드(랜딩 아님). 입력→분석→결과→export 흐름이 보여야 한다.
- 네이버 발행 금지(API 종료). 사람이 붙여넣어 임시저장만.
- 코드 전 `docs/designs/`에 설계 먼저(5단계 플로우). 모듈 상태 경계 침범 금지.
- 한국어 전용. 금지어 필터/콘텐츠 정책(2000자·12~32자·무번호목록) 준수.
- 민감정보(.env.local, 네이버 계정, 고객·매장 정보) 열람·출력 금지.
- 포트 3100. pnpm. vitest.
