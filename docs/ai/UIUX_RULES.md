## 이 프로젝트(업무형 파이프라인 대시보드) 적용
- **흐름이 명확해야 한다**: 매장 선택 → 키워드 → 본문 → 이미지 → export. 사용자가 지금 어느 단계인지 보인다(스테퍼).
- **첫 화면에서 다음 행동이 보여야 한다**(매장/카테고리 선택, 키워드 생성). 빈 대시보드 금지.
- **주요 CTA는 1~2개만 강조**. 버튼명은 행동 중심("키워드 생성", "본문 작성", "이미지 생성", "export 복사").
- 검증 결과(금지어/규칙 위반)는 눈에 띄게, 승인/수정 동선과 함께.
- 반응형·접근성(키보드/대비/ARIA)·다크모드(next-themes) 확인.
- 디자인 작업도 테스트(렌더 확인) 없이 완료라고 말하지 않는다.

## 상태 처리 (필수 3종)
- 로딩: 스켈레톤 권장(스피너 지양). AI 호출이 길 수 있으니 진행 표시 명확히.
- 빈 상태: 다음 행동 안내 포함.
- 오류: 원인 + 복구 행동(AI/네이버 API/타임아웃 실패 포함).

## 피해야 할 것
- 흔한 AI 그라데이션, 의미 없는 이모지, 카드 남발, 정체불명 버튼.
- 단계 흐름이 안 보이는 화면, 어디부터 눌러야 할지 모르는 대시보드.

## 방향성 레퍼런스
