# GEO v2 — 자연스러운 전체 재작성

v1.x까지 GEO는 "레거시 템플릿 블록 제거 + 단정 완화 + 표 추가(하단)"에 머물렀다. 실제 AI 검색 인용을 노리는 GEO는 본문을 자연스럽게 재작성해야 한다. v2는 이 간극을 메우기 위한 재설계다.

## 목적

생성된 네이버 블로그 본문을 **AI 검색 인용에 유리한 구조**로 자연스럽게 재작성한다. 단순 하단 추가가 아니라 본문 전체를 직조한다.

GEO가 노리는 AI 검색 대상: ChatGPT, Perplexity, Google AI Overview, Naver Cue:, Claude, Gemini 등.

## 트리거 조건

- 본문 생성 완료 후(Stage 2 이후) 사용자가 "GEO 최적화" 버튼을 눌렀을 때
- 자동 실행 아님(수동). 블로그 운영자가 결과를 검토한 뒤 적용 여부를 선택한다.

## 아웃풋 정의

1. `analyze` 모드: 현재 본문의 GEO 점수(100점) + 카테고리 4축 점수 + 추천 변경 사항(최대 5개)
2. `apply` / `start-advanced` 모드: 재작성된 본문 + before/after 점수 + 적용된 추천 id 목록

**재작성본의 형식적 요건**:

- 섹션 소제목 3~5개, 그중 **50% 이상은 질문형**(예: `## HIFU와 안면거상술은 어떤 층을 치료하나요?`)
- 섹션마다 **첫 줄에 40~80자 핵심 답변 한 문장**(AEO extract snippet 용도)
- 본문 중간에 **비교·기준 정리 목적의 markdown 표 1개**
- 단정 표현 없음(`100% 해결`, `반드시`, `무조건`, `완벽하게`)
- 의료·시력·건강 맥락은 "상황에 따라", "개인차", "상담" 신호로 완화
- 원문 대비 **주제·매장·키워드·업종이 모두 보존**
- 기존 도입부 2~3문단 톤은 유지하되, 첫 섹션 직전에 글 요지 1문장 추가 가능
- 하단 부속 블록(`## FAQ`, `## 확인 및 안내`) 금지 — 본문에 녹여 쓸 것

## 변환 5종 (recommendation IDs)

| id | 종류 | 영향도 | 기본 선택 | 비고 |
|----|------|--------|-----------|------|
| `remove-template-blocks` | 결정적 | high | O(아티팩트 있을 때) | 레거시 FAQ / 핵심 답변 / 확인 및 안내 제거 |
| `question-heading` | AI 재작성 | high | O | 설명형 소제목을 질문형으로 변환 |
| `direct-answer-lead` | AI 재작성 | high | O | 섹션 첫 줄에 40~80자 핵심 답변 삽입 |
| `comparison-table` | AI 재작성 | medium | 조건부 | 비교·기준·시기·방법 주제일 때 표 1개 삽입 |
| `soften-claims` | 결정적 | high | O(단정 감지 시) | 강한 확정 표현 완화 |

## 스코어링 재설계 (100점)

### AI 인용 구조 (30점)

- 질문형 소제목 비율: ≥50% → 12점, 30~50% → 8점, 10~30% → 4점, <10% → 0점
- markdown 표 존재 → 10점
- 레거시 아티팩트 없음 → 8점 (페널티가 아니라 가점)

### 신뢰성 & 근거 (25점)

- 단정 표현 없음 → 10점
- 출처·기준 언급 2회 이상 → 8점 (1회 4점)
- 개인차·상황 신호 → 4점
- 레거시 아티팩트 없음 → 3점

### 엔티티 & 지역성 (20점)

- 매장명 · 업종 · 기준일(`2026-04-19` 형식) 각각 5점 × 3 = 15점
- 레거시 아티팩트 없음 → 5점

### 본문 완성도 (25점)

- 글자수 1800 이상 → 8점 (1400 이상 6점, 그 외 3점)
- **섹션별 직답 라인 비율** ≥50% → 8점 (30~50% → 5점, <30% → 2점)
- 키워드 3종 모두 포함 → 5점
- 레거시 아티팩트 없음 → 2점
- 검증 이슈 없음 → 최대 2점

## 변환 파이프라인

### 결정적 변환 (fast path)

1. 레거시 아티팩트 제거 (`removeTemplateBlocks`)
2. 단정 표현 치환 (`softenClaims`)

### AI 재작성 (slow path, Claude sonnet-4.6)

사용자가 `question-heading` 또는 `direct-answer-lead` 또는 `comparison-table` 중 하나 이상 선택한 경우 실행.

프롬프트는 **선택된 변환 목록을 받아** 요구 사항을 동적으로 구성한다. 예:

- 질문형 변환 선택 시: "소제목 4~5개 중 3개 이상은 `~인가요?`/`~하나요?` 질문형으로 재작성"
- 핵심 답변 선택 시: "각 섹션 첫 줄에 40~80자 직답 한 문장"
- 비교 테이블 선택 시: "본문 중간에 3~4열 markdown 비교표"

**재작성 무결성 검사** (기존 유지):

- 원문 토큰 55% 이상 보존
- 매장명·업종 보존
- 제목 수정 금지
- 금지 패턴(`## FAQ`, `핵심 답변:` 리터럴) 재등장 차단
- 점수가 baseline 이하로 떨어지면 폐기

### 글 유형별 가드

가격표형·제품 소개형 포스트는 AI 재작성 경로를 **전면 차단**한다(레거시 아티팩트 제거와 단정 완화만 허용). 유형 판정 휴리스틱:

- 가격표형: `가격` · `원` · 표 3개 이상 · 글자수 <1200
- 제품 소개형: `입고` · `신제품` · `컬렉션` · 제품명 2개 이상

## 실패 복구

| 실패 모드 | 대응 |
|-----------|------|
| Claude API 타임아웃(60초) | baseline(결정적 변환만) 반환 |
| API 키 없음 | 결정적 변환만 실행 |
| 재작성본 무결성 검사 실패 | baseline 반환, 재작성 폐기 |
| 재작성본 점수가 baseline 이하 | baseline 반환 |
| 재작성본이 제목 라인을 복제 | `sanitizeGeoRewrite`에서 제거 |
| 재작성본이 `## FAQ` 재삽입 | 후처리로 다시 제거 |

## 상태 경계

v1.4와 동일 — GEO 모듈은 무상태. 분석·적용 모두 요청-응답 사이클 안에서 완결.

| 모듈 | 소유 상태 | 공유 인터페이스 |
|------|----------|----------------|
| `lib/geo/harness.ts` | 없음 | `runGeoHarness`, `applyGeoRecommendations` |
| `lib/prompts/geoRewritePrompt.ts` | 없음 | `buildGeoRewritePrompt({ article, selectedIds, targetScore })` |
| `app/api/article/geo/route.ts` | advancedGeoJobs(인메모리) | POST 핸들러 |

## Codex가 남긴 회귀 (v1.x → v2)

1. analyze 모드에서 `normalizeArticleForGeo`가 레거시 아티팩트를 **몰래 먼저 제거한 뒤** 점수를 냄 → 실제 본문은 아직 오염되어 있는데 "수정할 것 없음"으로 보임. (안경렌즈 스크래치 글 사례)
2. `geoRewritePrompt.ts`가 질문형 소제목 변환을 "금지 패턴"에 올려둠 → 실제 요구되는 GEO 변환이 프롬프트 수준에서 차단됨.
3. 재작성이 본문 전체를 자연스럽게 직조하지 않고 하단에 `## FAQ` / `## 확인 및 안내` 블록만 덧붙임 → 안경원 글 흐름이 깨짐.

v2는 위 3개를 모두 해결한다.

## 점진적 배포

1. 설계 확정 (이 문서)
2. 하네스 스코어링 재설계 + recommendation 5종으로 확장
3. 프롬프트 재작성 (질문형·핵심 답변·비교표 요구, 하단 추가 금지)
4. route.ts에서 사전 제거 로직 삭제, AI 재작성을 primary path로
5. 기존 발행 완료 글 중 레거시 아티팩트 남은 샘플로 회귀 검증
