# 자연어 다듬기 패스 (Naturalness Polish Pass)

## 목적
생성 본문에 남는 **어색한 압축 조어**("시선 자리"·"피팅 자리"류 — 명사+명사를 억지로 붙여 사람이 입으로 안 하는 말)를 사람 말투로 푸는 1회 LLM 패스. **하드코딩 블랙리스트 없이** LLM의 "소리 내어 읽기" 판정으로 부류 전체를 잡되, 진짜 안경 용어·키워드·구조는 보존한다. (배경: 생성 프롬프트의 소프트 원칙만으로는 100% 강제가 안 돼 가끔 조어가 샘 — 판정을 강제하는 단계가 필요.)

## 트리거 조건
본문 생성 → 자동 재수정 → G3(자완 보강)까지 끝난 뒤 **자동 1회 실행**. 단 `maxDuration(360s)` 잔여 시간이 `NATURALNESS_MIN_REMAINING_MS` 이상일 때만(응답 잘림 방지). 부족하면 스킵(생성 프롬프트 원칙은 그대로 유지).

## 실패 복구
- 타임아웃/오류 → 직전 본문 그대로 유지(graceful, 빈 catch).
- 결과가 **검증 후퇴**(키워드 누락·분량 이탈·하드 이슈)면 채택 안 함 → 원본 유지. (G3 보강과 동일한 회귀 방지 패턴: `needsHardRevision`이 false일 때만 채택)

## 상태 경계
- 본문 문자열만 다룬다. 키워드(메인/서브 원형)·매장 안내 블록·어미 계열·쉼표 금지 불변.
- 다른 모듈과 공유 상태 없음. `claude.ts reviseArticle`(EDIT_MODEL=Opus) 재사용, 새 클라이언트 없음.
- 프롬프트는 `revisionPrompt.ts buildNaturalnessPolishPrompt`(기존 보강 프롬프트와 동거).

## 아웃풋 정의
다듬어진 본문 `string`. 채택 시 `article.content` 대체 + `validation` 갱신. 어색한 조어가 없으면 모델이 원문을 그대로 반환(무변경).

## 비하드코딩 원칙
- 단어 목록을 만들지 않는다. 판별 기준은 "안경사가 손님 앞에서 입으로 말할 표현인가"(read-aloud).
- "누진대·단초점·굴절력·아이포인트·도수 구간" 같은 **실제 용어는 명시적으로 보존**(장황하게 풀지 말 것 — 장황함도 AI 티).
- 멀쩡한 문장은 건드리지 않는다(과교정 금지).
