# GEO Harness Engineering

## 목적

블로그 본문 생성 이후에 기존 생성 품질을 깨지 않으면서, GEO(Generative Engine Optimization) 관점의 후처리와 검증을 수행한다.

핵심 원칙:

- 생성과 최적화를 분리한다.
- 점수화 근거를 코드로 남긴다.
- 수정은 작은 수술식 패치로 적용한다.
- 적용 전/후 점수를 비교해 피드백 루프를 만든다.

## 적용한 전략

### 1. Harness Engineering

- `src/lib/geo/harness.ts`를 단일 하네스로 두고 분석과 적용을 모두 이 레이어에서 수행한다.
- GEO 평가는 휴리스틱 기반의 결정적 규칙으로 계산한다.
- 적용 결과는 항상 `analysisBefore` / `analysisAfter`로 반환한다.

### 2. Superpowers / Best Practice 계열에서 가져온 점

- 거대한 전역 규칙 대신, 현재 태스크에 직접 연결되는 작고 검증 가능한 후처리 단계로 분리
- 한 번에 전체 재생성하지 않고 질문형 소제목, 표, FAQ, 출처 섹션 같은 작은 변경만 적용
- 사람이 결과를 보고 선택 적용할 수 있는 UI 제공

### 3. Memory / Learning 계열에서 가져온 점

- 세션 복원 시 저장된 글에도 GEO 분석을 다시 수행
- 수동 편집 이후에도 GEO 분석을 재실행해 최신 상태를 유지

## 현재 점수 축

- AI 인용 구조
- 신뢰성 & 출처
- 엔티티 & 저자
- 콘텐츠 품질

- 비교 테이블 추가
- 의료법 저촉 가능 표현 완화
- 섹션별 핵심 답변 추가
- FAQ 추가
- 출처 및 확인 포인트 추가
- 작성 주체 / 업데이트 정보 추가

## 안전장치

- 기존 본문 생성 API는 그대로 유지한다.
- GEO는 생성 후 별도 API(`/api/article/geo`)에서만 동작한다.
- 적용 후에는 기존 `validateContent`를 다시 호출해 검증 결과를 갱신한다.
