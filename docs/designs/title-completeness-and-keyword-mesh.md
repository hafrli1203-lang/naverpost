> (D = 파워링크 CTR→본문 형식, PDF 정독 후 사용자 "업데이트 다 해라"로 추가 구현. 상세 §8.)

## 구현 로그 (2026-06-15)
- **A (제목 완성도, titlePrompt v23)**: `buildTitlePolishPrompt`·`TITLE_PATTERN_GUIDE`·`buildSeedTitlePrompt`에서 종속절 단독 종결 금지, 중립 약속어(차이/종류/기준/방법/이유/정리/장단점/뜻/순서/점) 마무리 재허용, [상황]+[약속] 2부 구조, 끝맺음 분산. 후기·가격·비교·추천·TOP은 약속어로도 금지 유지. `route.ts` 캐시 v24→v25.
- **B (그물망, keywordMesh.ts)**: 죽어있던 `buildTrendOptions` 안전 부활(실 수식어 요즘/2026/데일리/신상/첫 × head). 안경이야기에 스타일·경험 군집 시드(`GLASSES_STORY_LIFESTYLE_OPTIONS`) 추가 — 기존 조합 불변, 추가만. 선별 로직(`canAddIntentBalancedCandidate`)은 이미 같은 소재/테마/축 1개 초과를 강하게 거부하므로 손대지 않음(쏠림 원인은 선별이 아니라 후보 풀 빈약이었음).
- **C (코퍼스 깊게 쓰기, titleCorpus.ts)**: read-time `refineCorpusTitles` 추가 — 도메인 필터(안경 관련)·정책 필터(후기/가격/추천 제거)·지점명 제거·비안경 맥락(임장/갤럭시 등) 제거 후 **약속어 마무리 제목 우선 정렬**. 캐시 원본은 보존, 읽을 때 정제 → 생성·폴리시 양쪽 호출부가 자동 개선.
  - **검증 중 발견**: 코퍼스에 도메인 외 오염(정수기 렌탈·고양이 화장실·도수치료·부동산 임장·갤럭시)이 다수 섞여 있었음(수집 시드가 광범위). 필터로 read-time 차단. (근본 해결은 harvest 시드 정교화 — 후속 과제.)
  - 실데이터 결과: 카테고리당 clean 58~202개, 완성형(약속어끝) 5~36개. 예: "압축렌즈 굴절률 높이면 가장자리 두께가 줄어드는 이유", "기성돋보기와 맞춤돋보기 차이".

> 원래 상태: 설계(STEP 1). 코드 변경 전 사용자 승인 대기.
> 발단: 생성 제목이 "쓰다 만 것 같다"(종속절로 끊김) + 키워드/소재가 한 군집에 쏠려 다양성 부족.

## 0. 진단 (왜 이렇게 나오나)

### 0.1 제목이 "쓰다 만 것 같다"
- `titlePrompt.ts > buildTitlePolishPrompt`(v22)가 **약속 명사 마무리(~법/~순서/~점/~부분/기준)를 금지**하고 "~한다면/~볼까/~괜찮을까" 조건·질문형으로 바꾸라고 강제.
- 한국어에서 "~신경 쓰인다면", "~궁금하다면"은 **종속절(미완성 문장)** → 글자 그대로 끊김.
- 동시에 `절대 금지 단어`(후기·가격·비교·추천·TOP)가 끝맺을 "정보 약속어"를 모두 제거 → 제목이 웅얼거리다 종료.
- v22는 명사 **스택**("맡기기 전 확인 순서 점검")을 잡으려던 것인데 약속 명사 **자체**를 막아 과교정됨.

### 0.2 실제 네이버 안경원 제목의 구조 (직접 수집)
glassesbaksa(신림 난곡점), optician-gil(수원 망포점)에서 수집:
- "안경 코받침 자국, 왜 심하게 남을까? 방지 & 피팅법 완벽 정리"
- "누진 다초점 vs 오피스 렌즈 차이점 정리, 나에게 딱 맞는 안경은?"
- "둥근? 긴? 각진? 각 얼굴형에 맞는 안경테 고르는 방법"
- "안경 처방전 보는 법, 읽는 법 - SPH, CYL, AXIS, PD 뜻"
- "\"치약으로 닦으면 된다?\" 안경 렌즈 기스 제거의 진실"
- "난시 안경 어지러움, 맞게 맞췄는데 왜 계속 불편할까요?"
- "변색렌즈 장단점, 틴트 변색렌즈 차이 - 한 번에 정리해드립니다"

**공통 DNA(= 우리 제목에 없는 것):**
1. **약속/payload 명사로 끝맺음**: 정리 / 방법 / 법 / 차이(점) / 장단점 / 이유 / 진실 / 뜻
2. **2부 구조**: [공감 상황·구체 소재] + 구분자 + [구체 약속·반전 질문]
3. **구체 디테일**: 숫자(3가지·50대), 분류(둥근/긴/각진), 대조(vs)
4. **반전·인용 훅**: "설마 이렇게 닦으세요?", 통념 깨기("치약으로 닦으면 된다?")
5. 쉼표·물음표·대시 자유 사용

우리 제목은 위 중 **(1) 약속어가 없고 (2) 상황 절반만** 있어서 끊긴다.

### 0.3 키워드/소재 쏠림
- `keywordMesh.ts` `glasses-story` 축이 관리·수리 중심(수리/세척/관리/보관/피팅/흘러내림/김서림).
- `buildTrendOptions()`가 빈 배열(303줄, 사실상 비활성).
- 조합이 대부분 `head × situation` 단일 축 → 그물망이 아님.
- `route.ts` 최종 10개에 **군집 분산 쿼터 없음** → 라이브 생성물이 한 군집에 몰리면 그대로 통과.

## 1. 목적
"실제 검색자가 쓰는 키워드"를 **교차 조합으로 대량 생성**하고, 그 키워드로 **완성된(약속어로 끝맺는) 사람 말투 제목**을 만든다. 의료법/저품질 회피는 유지한다.

## 2. 트리거 조건
기존과 동일: 매장+카테고리 선택 시 `POST /api/keywords`. 새 트리거 없음.

## 3. 변경 범위 (3블록)

### 블록 A — 제목 완성도 복구 (titlePrompt v23)
정책 결정 반영: **중립 약속어만 허용**(가격·후기·비교·추천·TOP 계속 금지).
- 허용 약속어(끝맺음): 차이 / 종류 / 기준 / 방법 / 이유 / 정리 / 장단점 / 뜻 / 순서 / 점.
- **금지 유지**: 명사 **스택**(2개 이상 연속 "확인 순서 점검"), 후기·가격·비교·추천·TOP·충격·필독.
- **종속절 단독 종결 금지**: "~신경 쓰인다면"으로 끝나면 실패 → 약속어 절을 붙이거나 완결 질문/평서로.
- **끝맺음 분산**: 한 후보군에서 같은 끝맺음 계열 3개 초과 금지(약속명사형/완결질문형/상황평서형 섞기).
- 2부 구조 권장: [상황·소재] + [약속어/반전질문]. (예: "안경 코받침 자국이 남는 이유와 줄이는 방법")
- 쉼표는 제목 정책상 무콤마 유지(연결어미로 두 절을 잇기) — CLAUDE.md 정책 충돌 없음.

### 블록 B — 키워드 그물망 실효화
- `buildTrendOptions` 부활(시즌·트렌드 축 실제 조합).
- 교차 축 확대: `head × situation` 외 `대상 × 상황`, `소재 × 상황`, `트렌드 × head` 추가 결합.
- `glasses-story` 축에 스타일/경험/상황 군집을 생성 단계부터 투입(현재는 fallback에만 있음).
- 최종 선별에 **군집 분산 쿼터**: 카테고리별 소재 군집(관리/스타일/상황/경험/트렌드 등) 최소 1개씩 강제.
- **검색량 게이트 유지·강화**: 생성된 교차 후보를 `fetchKeywordDemandSignals`로 거르고, 마광수 원칙대로 **롱테일(검색량 낮음·경쟁 낮음·전환 높음=황금키워드) 우선** 정렬. (0볼륨 제거)

### 블록 C — 실제제목 코퍼스 "깊게 쓰기" (정확도·사람글)
**정정: 코퍼스 인프라는 신규 구축이 아니다. 이미 구축·작동 중이다.**
- 이미 있는 것:
  - `lib/analysis/titleCorpus.ts` — 네이버 **공식 블로그 검색 API**로 카테고리별 실제 제목 최대 300개 수집, 30일 캐시·TTL·실패 graceful 폴백까지 완성.
  - `data/title-corpus.json` — 146KB, `lenses` 등 실제 제목 수집 완료(2026-06-12). (빈 깡통 아님)
  - `route.ts` → `getCorpusTitles` → `buildTitlePolishPrompt`에 "[실제 업종 제목 용어 기준]"으로 **이미 주입 중**.
- 부족분(= 이번 작업): 데이터는 모았는데 **얕게만 쓴다.** 지금은 폴리시 단계에 "이 정도 용어 수준을 따르라"는 막연한 참고로만 닿음.
  1. **구조 템플릿 추출**: 코퍼스에서 `[상황]+[약속어]` 패턴·끝맺음 어휘 풀(정리·방법·차이·이유…)을 분해.
  2. **생성 단계 few-shot 주입**: 폴리시(다듬기)뿐 아니라 제목을 *만드는* 프롬프트에 in-domain 예시로 투입.
  3. **정책 필터**: 원천 제목에 후기·가격·지역·매장명이 섞여 있음(현 필터는 이벤트/할인만 거름) → 그대로 예시화하면 금지 각도 역유입. **구조만** 뽑아 쓰고 표면어는 거른다.
- Playwright 우회는 **코퍼스 수집용이 아니다**(수집은 공식 API로 이미 됨). 검색 API가 안 주는 **특정 경쟁 블로그 본문 정독**(사장님 지정 8개 블로그 등)에 쓰는 별개 보조 수단.

## 4. 실패 복구
- Playwright 수집 실패/네이버 차단 변동 → 마지막 성공 코퍼스(`title-corpus.json`) 재사용, 없으면 기존 내장 규칙으로 폴백(현 동작 유지).
- 검색량 API 무응답 → 현 volumeGate 폴백 경로 유지(부분일치 폴백은 메모리 경고대로 재도입 금지).
- 그물망 과생성으로 후보 폭증 → `maxCandidates` 상한 + 군집 쿼터 후 절단.

## 5. 상태 경계 (충돌 방지)
- 변경 파일: `lib/prompts/titlePrompt.ts`(A), `lib/keywords/keywordMesh.ts`·`route.ts` 선별부(B), `lib/analysis/titleCorpus.ts` + 수집 스크립트(C).

## 6. 아웃풋 정의
- 카테고리당 후보 14개 생성 → 검색량·군집 쿼터·자기잠식 가드 후 **10개**, 각 제목은 약속어로 끝맺고 끝맺음 계열이 섞임.
- 다양성 수용 기준(검증): 최종 10개의 소재 군집이 **3종 이상**, 동일 끝맺음 계열 ≤3, main 키워드 중복 0.

## 7. 미해결/정직한 한계
- "확실히 대량으로"는 **검색량 API 커버리지**에 의존. 그물망 생성은 조합적으로 수백 후보를 만들 수 있으나, 실검색 검증을 통과하는 황금키워드 수는 API가 잡아주는 만큼이 상한. 과약속 금지.
- 실제 상위 제목 다수는 **고객 사례·가격·후기 각도**라 의료법/정책상 그대로 차용 불가 — 구조·약속어·훅만 차용한다.

## 8. PDF "팔리는 키워드의 비밀" 정독 요약 (2026-06-15)
저자(권쎄오)의 방법론. 우리 코드와 대조:

**우리 구조를 확증(이미 구현됨):**
- 그물망 키워드 소싱 5+1법: ①지역/랜드마크(동의어·대체어) ②메뉴=상품 ③소비상황 ④트렌드·시의성 ⑤형용사·수식어 ⑥오타·동의어 → 나열 후 조합. → 우리 `keywordMesh` 축(heads/materials/situations/targets/features/trendTerms)이 그대로 대응.
- 황금키워드 = "검색량 대비 발행량(경쟁) 낮은" 키워드. 90% 시간을 키워드 찾기에. → 우리 volumeGate·opportunityScore.
- 욕구키워드(대표·조회↑전환↓) → 수요키워드(세부·조회↓전환↑). 전환은 세부에서. 롱테일 우선. → 메모리 exposure_analysis·v20과 일치.
- 키워드 3부 조합: 대표 + 메뉴(상품) + 호기심 키워드. → 우리 main + sub1 + sub2.
- 유효성 검토 필수(검색량 0 제거). 벤치마킹=상위 1~5등 키워드 개수 평균 반영. → competitorMorphology/smartBlock 부분 구현.

**제목 관련 확증:**
- 상위노출 사례: 제목의 모호어("A to Z")를 구체어로 교체 + 공감/스크랩/댓글 반응 → 20등→1등. **제목 구체성**이 노출에 직결. → 우리 v23(약속어·완성도) 방향과 정확히 일치.

  - CTR은 이미 신호에 있음(`SearchVolumeSignal.monthlyAverage{Pc,Mobile}Ctr`). article route 병렬 배열에 `fetchKeywordDemandSignals([main])` 추가(graceful) → 모바일/PC 평균 CTR 최대값이 **1% 이상=info-first(핵심 먼저, 도입 짧게), 미만=experience-first(장면 공감 먼저)**, 신호 없으면 미지정(현재 동작).
  - 흐름: `route.ts`(CTR 페치·산출) → `articleBrief.ts`/`types`(`contentFormat` 필드) → `articlePrompt.ts`(`contentFormatSection` 주입). 의료광고 회피상 후기형은 안 쓰고, 도입부 3파트 구조는 유지하되 **핵심 배치/공감 길이만 조절**. 타입체크 통과.
- 검색동선·전환동선 우선순위: 세부키워드(검색동선 짧음)+근거리(전환동선 짧음)=1순위, 데이트/놀거리류(둘 다 긺)=후순위. → 우리 intent bucket(problem/selection/situation 우선)과 부합. 추가 튜닝 여지.
- **학술 논문(arXiv 등) 추출은 이 프로젝트에 없다.** 코드 확인 결과:
