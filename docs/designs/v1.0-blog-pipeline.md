# 설계 문서: v1.0 블로그 자동 작성 파이프라인

## 목적
6개 안경원 네이버 블로그의 콘텐츠를 자동 생성하고 임시저장까지 수행하는 웹 대시보드.

## 트리거 조건
- **수동 실행**: 사용자가 웹 대시보드에서 매장/카테고리/주제를 선택하고 "시작" 버튼 클릭
- **Human-in-the-loop**: 각 단계(키워드 선택, 본문 확인, 이미지 확인, 최종 컨펌)에서 사용자 승인 필요

## 실패 복구

| 실패 지점 | 처리 방법 |
|-----------|----------|
| Claude API 오류 | toast 에러 메시지 표시, 사용자가 재시도 가능 |
| Perplexity API 오류 | toast 에러 메시지 표시, 재시도 가능 |
| Google AI 이미지 생성 실패 | 개별 이미지 "failed" 상태, 개별 재생성 버튼 제공 |
| SSE 연결 끊김 | `onerror`에서 연결 종료 후 에러 토스트 표시 |
| Naver 토큰 만료 | `withTokenRetry()`가 자동 갱신 후 1회 재시도 |
| Naver API 오류 | 에러 메시지 반환, 사용자가 재시도 가능 |
| 브라우저 종료/새로고침 | `usePersistedWorkflow`가 localStorage에 상태 자동 저장, 복구 |

## 상태 경계

### 서버 사이드 상태
- **이미지 파일**: `os.tmpdir()/naverpost-images/{sessionId}/` (24시간 후 자동 정리)
- **토큰 저장소**: `tokenManager.ts` 인메모리 (프로세스 종료 시 초기화, 환경변수에서 재로드)
- **충돌 가능성**: 없음. 각 세션은 `sessionId`로 격리됨

### 클라이언트 사이드 상태
- **워크플로우 상태**: localStorage `naverpost-workflow` 키
- **충돌 가능성**: 탭 2개 동시 사용 시 충돌 가능 → 단일 탭 사용 전제

### 모듈 간 의존 관계
```
page.tsx (오케스트레이터)
  ├─ /api/keywords ─→ claude.ts ─→ titlePrompt.ts
  ├─ /api/article ──→ perplexity.ts + claude.ts ─→ articlePrompt.ts + contentValidator.ts
  ├─ /api/image/generate ──→ claude.ts + imageGen.ts + imageStore.ts
  ├─ /api/image/regenerate ─→ imageGen.ts + imageStore.ts
  ├─ /api/image/file/[imageId] ─→ imageStore.ts
  └─ /api/publish ──→ tokenManager.ts + blogApi.ts + contentFormatter.ts
```

## 아웃풋 정의

| 단계 | 아웃풋 형태 | 전달 경로 |
|------|-----------|----------|
| 키워드 생성 | `KeywordOption[]` (3개) | JSON → UI 카드 표시 |
| 본문 작성 | `ArticleContent` (제목+본문+검증결과) | JSON → UI 미리보기 |
| 이미지 생성 | `BlogImage[]` (최대 10개) | SSE → UI 그리드 |
| 임시저장 | `{ postId: string }` | Naver 블로그 임시저장함 |
