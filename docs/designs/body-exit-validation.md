# 본문 출구 검증 v1.0 — 사람화 규칙의 결정론적 보증

## 목적
사람화 규칙(AI 상투어 금지·쉼표 금지·어미 3연속 금지·문두 접속사 ≤3)이 프롬프트
지시로만 존재해, LLM이 어기면 그대로 통과하는 구멍을 막는다. 출구에서 결정론적으로
검사해 위반 시 기존 자동 수정 루프(reviseArticle)로 되돌린다.

## 트리거 조건
- validateContent 호출 시 항상 (생성 직후 검증 + 워싱 후 washReport 동일 적용).

## 검사 항목과 강도
| 항목 | 판정 | 강도 |
|---|---|---|
| 강한 AI 상투어 (이처럼/이러한/이를 통해/뿐만 아니라/알아보겠습니다/살펴보겠습니다/이상으로/도움이 되셨길/참고하시기 바랍니다/첫째,둘째 나열) | 1회라도 검출 | **하드** (수정 루프 발동) |
| 본문 쉼표 (표 행·숫자 천단위 제외) | 3회 이상 | **하드** |
| 동일 어미 3연속 (문장 끝 3음절 키 기준) | 검출 | 소프트 (mechanicalSignals — 수정 시 지시문 포함) |
| 문두 접속사(그리고/그래서/하지만/또한/따라서/그런데) | 4회 이상 | 소프트 |

기존 aiCliches(차근차근 등 모호한 목록)는 소프트 유지 — 과차단 방지.

## 실패 복구
- 수정 루프는 기존 MAX_REVISION_ATTEMPTS 상한 그대로 (무한 루프 없음).
- 검사 자체는 순수함수 — 외부 의존·실패 경로 없음.

## 상태 경계
- contentSignalAnalyzer(순수함수)에 검출 추가, LanguageRiskAnalysis에
  strongAiCliches/formatViolations 필드 추가(옵셔널 — 기존 호출 비파괴).
- needsHardRevision(article route)과 revisionPrompt에 연결.
