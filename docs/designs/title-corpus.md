# 실제 업종 제목 말뭉치 (Title Corpus) v1.0

## 목적
손수 만든 규칙·동의어 맵(두더지잡기)을 대신해, 네이버 공식 검색 API로 안경 업종의
실제 상위 블로그 제목을 카테고리별 수백 개 수집해 "용어·표현·분류의 기준"으로 삼는다.
생성 LLM은 말뭉치에 없는 생소한 조어·분류(예: "고굴절" 뭉뚱그림, "눈피로 안경렌즈" 병렬)를
만들 수 없게 되고, 검수(폴리시)는 말뭉치를 용어 사전처럼 참조한다.

## 트리거 조건
- 키워드 생성 요청(POST /api/keywords) 시 카테고리 말뭉치를 조회.
- 캐시가 없거나 30일 경과 시 lazy 수집(해당 요청에서 1회, 이후 캐시).

## 수집 설계
- 소스: 네이버 OpenAPI 블로그 검색(`/v1/search/blog.json`, sort=sim) — 공식 API, 스크래핑 아님.
- 쿼리: 카테고리명 + 세부주제 + BROAD 헤드 키워드 (~20쿼리/카테고리, display=30).
- 호출량: 카테고리당 ~20회, 6개 전체 월 ~120회 (일 한도 25,000회 대비 무시 가능).
- 필터: 길이 8~40자, 광고성 단어(이벤트/할인/최저가/특가/문의/예약) 제외, 중복 제거.
- 저장: `data/title-corpus.json` — `{ version, categories: { [id]: { harvestedAt, titles[] } } }`
  카테고리당 최대 300개 보관.

## 활용 (아웃풋 정의)
1. GPT 후보 생성 프롬프트: 상위 40개를 "[실제 업종 상위 제목 말뭉치]" 섹션으로 주입.
   지시: 용어·분류·표현 수준의 기준으로 삼되 문장 복사 금지, 말뭉치에 없는 조어 금지.
2. Claude 편집 프롬프트: 동일 섹션 주입.
3. 폴리시(사실성 검수): 15개를 용어 기준 목록으로 주입.

## 실패 복구
- API 실패/자격증명 없음 → 빈 말뭉치 반환, 키워드 생성은 기존 흐름대로 진행(graceful).
- 수집 실패 시 기존 캐시(만료됐어도) 유지 사용.
- 캐시 쓰기 실패는 무시(생성 차단 금지). 죽은 신호는 notes로 표면화.

## 상태 경계
- 소유 상태: `data/title-corpus.json` (이 모듈만 읽고 씀).
- 공유 인터페이스: `getCorpusTitles({ categoryId, seedQueries, limit })` — 순수 조회.
- 기존 `fetchCompetitorTitles`(요청별 경쟁 제목)와 별개: 그쪽은 차별화용, 이쪽은 표현 기준용.
