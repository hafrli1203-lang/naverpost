# 설계 문서: v1.1 안경원 관리 + 자동 주제 추천

## 목적
안경원을 UI에서 등록/수정/삭제하고, 등록된 데이터로 RSS 중복 방지 및 AI 자동 주제 추천을 수행한다.

## 트리거 조건
- **안경원 관리**: 사용자가 관리 페이지(`/admin`)에서 수동으로 CRUD 수행
- **자동 주제 추천**: 메인 페이지에서 매장 + 카테고리 선택 시 AI가 자동으로 주제 3개 추천
- **수동 주제 입력**: 기존처럼 직접 주제를 입력하는 것도 가능 (선택사항)

## 실패 복구

| 실패 지점 | 처리 방법 |
|-----------|----------|
| shops.json 파일 읽기 실패 | 빈 배열 반환, 에러 토스트 |
| shops.json 파일 쓰기 실패 | 에러 메시지 반환, 기존 데이터 유지 |
| RSS 크롤링 실패 (개별 매장) | 해당 매장 스킵, 나머지 매장은 계속 진행 |
| AI 주제 추천 실패 | 수동 입력 폼으로 폴백 |

## 상태 경계

| 모듈 | 소유 상태 | 변경사항 |
|------|----------|---------|
| `data/shops.json` (신규) | 안경원 목록 (파일 저장) | 기존 constants.ts 하드코딩 대체 |
| `/api/shops` (신규) | 없음 (파일 I/O만) | CRUD API |
| `/api/topics/suggest` (신규) | 없음 | AI 주제 추천 API |
| `lib/constants.ts` | 카테고리만 유지 | SHOPS 배열 제거 |
| `rssParser.ts` | 없음 | shops.json에서 매장 목록 읽도록 변경 |

## 아웃풋 정의

| 기능 | 아웃풋 형태 | 전달 경로 |
|------|-----------|----------|
| 안경원 목록 조회 | `Shop[]` | GET /api/shops → UI |
| 안경원 추가 | `Shop` | POST /api/shops → shops.json |
| 안경원 수정 | `Shop` | PUT /api/shops → shops.json |
| 안경원 삭제 | `{ success: true }` | DELETE /api/shops → shops.json |
| 주제 자동 추천 | `string[]` (3개) | POST /api/topics/suggest → UI |
