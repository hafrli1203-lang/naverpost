# 네이버 붙여넣기용 안전 export v1.0

## 목적
네이버 글쓰기 API가 종료(2020)되어 자동 발행이 불가능하다. 사람이 스마트에디터에
붙여넣는 현재 방식을 유지하되, 그 마찰을 최소화한다. (계정 제재 리스크 없는 무위험 경로)

## 배경 (고친 문제)
기존 FinalConfirm "네이버 HTML 복사"는 `clipboard.writeText(html)` — HTML을 평문으로
복사해 스마트에디터에 붙이면 `<p>` 태그가 그대로 박힌다. 또 본문 이미지 src가
`/api/image/file/...` 상대경로라 네이버에 붙여도 안 뜬다.

## 동작
1. **본문 복사 (서식 유지)**: ClipboardItem으로 text/html + text/plain 동시 복사.
   - 리치 에디터(스마트에디터)는 text/html을 받아 제목·소제목·표·굵게가 살아서 붙는다.
   - 이미지 위치에는 `<img>` 대신 "[사진 N 자리]" 마커 블록을 넣는다(상대경로 깨짐 방지).
   - text/plain 폴백: ## 마커 제거, 표는 "셀 / 셀" 줄로, 줄바꿈 보존, [사진 N] 마커.
   - ClipboardItem 미지원 브라우저는 writeText(plain)로 폴백.
2. **이미지 전체 저장**: 성공 이미지를 순서대로 `블로그사진-1.jpg ...`로 일괄 다운로드
   (브라우저 다중 다운로드 차단 방지 위해 400ms 간격).
3. **4단계 안내 카드**: (1) 네이버 글쓰기 열기 (2) 본문 칸에 붙여넣기(Ctrl+V)
   (3) [사진 N] 위치에 저장한 이미지 끌어다 놓기 (4) 임시저장 버튼 클릭(발행 아님).

## 실패 복구
- 클립보드 권한 거부/미지원 → writeText 폴백, 실패 시 안내 토스트.
- 이미지 fetch 실패 → 해당 장 건너뛰고 계속(부분 성공 허용).
- 순수 포맷 함수(formatForNaverExport/buildNaverPlainText)는 외부 의존 없음.

## 상태 경계
- contentFormatter.ts에 순수함수 2개 추가(기존 formatForNaver 비파괴).
- FinalConfirm.tsx UI 교체(복사 버튼 동작 + 일괄 저장 + 안내). 새 의존성 없음.

## 정직한 한계
네이버 스마트에디터가 붙여넣기 시 일부 스타일을 제거할 수 있어 제목/표는 가벼운
손질이 필요할 수 있다. 안내에 명시한다. 이미지는 자동 삽입 불가(수동 드래그).
