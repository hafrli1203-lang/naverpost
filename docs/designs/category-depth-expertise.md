# 카테고리별 깊이·전문성 키워드 체계 (설계)

> 상태: 설계+구현 (2026-06-04). 참조: C:\project\blogposting 의 깊이 메커니즘.

## 목적
키워드(및 본문)가 "아큐브 건조 / 건조감 / 착용"처럼 표면 관리·세척어를 반복하는 문제를 없애고,
카테고리별로 **사실 기반 전문 각도**(스펙·메커니즘·기준 수치)로 다양하고 깊게 생성되게 한다.
blogposting의 3대 깊이 장치를 이식: ① 추상 라벨 금지 ② 전문 depth 차원 주입 ③ 근접중복 금지.

## 근거 (실측)
콘택트 생성 결과: "아큐브 건조/착용/건조감" 3개 근접중복 + 10개 전부 건조·착용·위생·관리.
산소투과율·함수율·베이스커브·토릭축·소재 같은 전문 깊이 0. → 표면 반복.

## 설계
1. **`src/lib/keywords/categoryDepth.ts`** — 카테고리별 전문 depth 차원(실제 광학 스펙/메커니즘) 단일 소스.
   차원 추가하면 자동 반영(확장성). 예: contacts = 산소투과율(Dk/t)·함수율·베이스커브·직경·토릭축·소재(실리콘하이드로겔)·단백질침전·각막산소·착용시간·UV차단.
2. **GPT 키워드 프롬프트(`openaiKeywords.ts`)에 주입**:
   - 위 depth 차원에서 서로 다른 것을 골라 생성, 한 차원에 최대 2개.
   - 범용 표면어(관리/세척/위생/생활 습관/사용감)만 반복 금지. 각 키워드는 구체 전문 포인트를 담을 것.
   - 같은 메인 핵심어를 미세변형 반복 금지(건조 vs 건조감).
3. 카테고리별 `category.id` → `CATEGORY_DEPTH_DIMENSIONS[id]`를 pool→GPT로 전달.

## 트리거/실패복구
- 키워드 생성 시 자동. depthDimensions 없으면(미정의) 기존 동작 유지(graceful).

## 아웃풋/수용 기준
- 콘택트 재생성 시 산소투과율·함수율·베이스커브·토릭 등 전문 각도가 섞이고 근접중복이 사라진다.
- type-check/lint 통과. 카테고리 정합성·개수 유지.

## 후속 (이번 범위 밖)
- 본문(articlePrompt)에도 동일 depth 차원 + 추상 라벨 금지 적용 → 본문 전문성 강화.
- 사실 정확도는 Perplexity 리서치(본문 단계 기존 보유)로 검증.
