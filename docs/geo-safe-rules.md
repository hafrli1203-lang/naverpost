# 안경원 GEO 안전 규칙

이 문서는 네이버 블로그용 안경원 글에서 GEO 후처리가 무엇을 해야 하고 무엇을 하면 안 되는지 정리한 기준표다.

목표는 두 가지다.

1. AI가 인용하기 불리한 기계식 흔적은 제거한다.
2. 네이버에 맞춰 이미 잘 작성된 본문 흐름은 깨지 않는다.

## 기본 원칙

- GEO는 본문을 다시 쓰는 기능이 아니다.
- GEO는 네이버형 글에 덧붙은 부자연스러운 템플릿 흔적을 걷어내는 보수적 후처리다.
- 제목, 도입부, 소제목, 표를 자동으로 갈아엎지 않는다.
- 수정 이득이 불분명하면 그대로 둔다.

## 제거 대상

다음 패턴은 실제 발행본 검토에서 반복적으로 확인된 기계식 GEO 흔적이므로 제거 대상이다.

- `핵심 답변:`
- `## FAQ`
- `## 확인 및 안내`
- `## 자주 묻는 질문`
- `## 참고 및 확인 포인트`
- `공개 자료와 현장 상담 관점을 바탕으로`
- `상담 관점에서 보면`
- `업데이트 기준일`
- 본문 맨 앞의 `...는 어떤 기준으로 보면 좋을까요? 현재 상태, 생활 패턴, 기대하는 변화 기준으로 나눠 보면 훨씬 이해가 쉬워집니다.`

## 보존 대상

다음 요소는 네이버형 안경원 글에서 자연스러운 구조로 확인되므로 기본적으로 보존한다.

- 설명형 제목
- 자연스러운 도입 2~3문단
- 정보 전달용 중간 표
- 매장 안내 문단
- 체크인, 지도, 위치 정보 전후 문장
- 제품 소개형 포스트의 입고 안내 문장
- 가격표형 포스트의 표와 가격 안내 문장

## 자동 비활성 대상

이 앱에서는 아래 자동 개입을 기본 비활성으로 유지한다.

- 질문형 소제목 자동 변환
- 도입부 직답 문장 자동 삽입
- FAQ 자동 생성
- 섹션별 핵심 답변 자동 생성
- 작성자 메타 블록 자동 삽입

이유는 네이버 글쓰기 규칙과 충돌하기 쉽고 실제 발행본 흐름을 망치기 때문이다.

## 허용되는 제한적 개입

아래 개입은 조건부로 허용된다.

- `remove-template-blocks`
  - 기계식 GEO 흔적 제거
- `soften-claims`
  - 건강/시력 관련 과장 표현 완화
- `comparison-table`
  - 비교, 시기, 기준 정리가 핵심인 글에서만 선택적으로 보강
  - 이미 자연스러운 표가 있으면 추가하지 않는다

## 글 유형별 기준

### 증상형

- 예: 시력저하, 야간시력 저하, 어지러움, 번짐
- 허용: 기준표, 검사 시기 표, 과장 표현 완화
- 금지: 맨 앞 질문형 한 줄 삽입, FAQ 추가

### 관리형

- 예: 안경관리 여름, 메탈안경 코팅 관리, 변색렌즈 수명
- 허용: 관리 기준 정리, 잘못된 핵심답변 블록 제거
- 금지: 도입부 재작성, 제품 홍보형 문장 일괄 삽입

### 비교형

- 예: 렌즈 A/B 차이, 편광렌즈와 일반 선글라스 차이
- 허용: 비교표 보강
- 금지: 소제목 전체 질문형 전환

### 가격표형

- 예: 월간 가격표, 품목별 가격 안내
- 허용: 과장 표현 완화
- 금지: 표 제거, FAQ 추가, 도입부 구조 변경

### 제품 소개형

- 예: 신제품 입고, 브랜드 소개, 프레임 추천
- 허용: 과장 표현 완화
- 금지: 정보형 GEO 블록 삽입, 본문 구조 재배열

## 테스트 기준

회귀 테스트는 최소한 아래를 계속 보장해야 한다.

- 기계식 GEO 흔적은 제거된다.
- 정상적인 네이버 도입부는 남는다.
- 가격표형 포스트는 건드리지 않는다.
- 제품 소개형 포스트는 건드리지 않는다.
- 비교형/비용형 글에서는 표 보강 추천이 가능하다.

## 운영 메모

- 실제 발행본을 계속 샘플링해서 제거 대상 패턴을 확장한다.
- 새 패턴을 추가할 때는 반드시 보존 테스트도 같이 추가한다.
- 규칙 추가보다 오탐 방지가 우선이다.
