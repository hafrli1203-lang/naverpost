# 사람이 쓴 글 메타 원칙

> 출처: `자료/잘 쓴 의료 블로그 100선 분석.docx` + `자료/팔리는 블로그 글쓰기 공식 & 패턴 공략집.docx`
> 활용: 본문 작성 프롬프트에 동적으로 주입되어 모델이 통계적 평균이 아닌 사람 글의 결을 모방하도록 유도

## 1. 글의 본질

블로그 글은 **정보 제공**이 아니라 **독자 의사결정 문서**다.
독자가 다음 결론에 도달하도록 만든다.

- 내 문제를 이 글이 정확히 이해한다
- 이 분야에 경험과 근거가 있다
- 이 방법이 실패 확률을 낮춘다
- 다음에 무엇을 챙기면 되는지 안다

## 2. 도입부 4줄 공식 (필수)

도입부 3문단 안에 다음 네 요소가 자연스럽게 녹아 있어야 한다.

1. **문제**: 이 글이 해결하는 상황 (대상·증상·시점)
2. **손실**: 방치 시 생기는 불편 또는 놓치는 것
3. **자격**: 이 글이 말할 만한 근거 (관찰·경험·기준)
4. **얻는 것**: 끝까지 읽으면 챙길 정보 (체크리스트·기준·확인 포인트)

도입부 첫 문장은 다음 다섯 패턴 중 하나로 시작한다 (반복 패턴 회피).

- 구체 장면형: "퇴근하고 화면을 오래 본 날, 글자가 미묘하게 흐릿하게 보일 때가 있어요."
- 일반 진술형: "안경테는 얼굴 한가운데 자리를 차지하는 물건이에요."
- 인용 따옴표형: "안경이 자꾸 흘러내려요. 이런 말씀을 듣는 일이 한 주에도 여러 번 있어요."
- 질문 환기형: "오후가 되면서 코받침 자국이 평소보다 진해졌다고 느낀 적 있으세요?"
- 직업 경험형: "상담을 하다 보면 같은 안경을 쓰는데 어떤 분은 편하고 어떤 분은 불편하다고 하시는 경우가 자주 있어요."

도입부 첫 문단을 제품 정의·구조 설명·기관 인용으로 시작하지 않는다.
"많은 분들이", "이번 글에서는", "차근차근", "한 번쯤", "바쁜 일상" 같은 AI 상투어를 쓰지 않는다.

## 3. 본문 — 원칙·이유·사례·적용 4단

각 소제목 아래는 다음 4단을 자연스럽게 풀어 쓴다. 네 요소를 같은 순서로 기계적으로 반복하지 말고 글 흐름에 맞게 변형한다.

- **원칙**: 결론 한 줄 (예: "코받침 위치가 바뀌면 시야 중심도 같이 바뀝니다")
- **이유**: 광학·해부학·생활환경 근거
- **사례**: 독자가 즉시 떠올릴 수 있는 생활 장면
- **적용**: 지금 확인할 수 있는 행동 한두 가지

본문은 설명이 아니라 **반박 처리**다. 독자의 의심 5개를 미리 푸는 구조로 짠다.
대표 반박: "다 비슷해 보이는데", "비싸지 않나", "지금 안경도 괜찮은데", "내 얼굴엔 안 어울릴 것 같은데", "관리가 귀찮을 것 같은데".

## 4. 어미 다양성 — 한 글 안 6~9종

100선의 잘 쓴 글은 한 편에 어미 6~9종이 자연스럽게 분포한다. 같은 어미 연속 반복(2회 이상) 금지.

- 표준 어체 예: "있습니다 / 좋습니다 / 가능성이 큽니다 / 영향을 줄 수 있습니다 / 주목받고 있습니다 / 어렵게 합니다 / 필수적입니다 / 인지요?"
- 친근 어체 예: "있어요 / 예요 / 이에요 / 네요 / 거든요 / 죠 / 답니다 / 봐요"
- 캐주얼 어체 예: "있어요 / 예요 / 네요 / 죠 / 봐요 / 거든요"

같은 단어를 본문 전체에서 2번 이상 반복하지 않는다.
"수월하다 / 확인할 수 있다 / 도움이 된다 / 쉽다 / 좋다" 같은 평이한 표현은 첫 등장 후 다른 어휘로 풀어 쓴다.
대체 어휘 예: 수월하다 → 편하다·살피기 좋다·기준 잡기 쉽다·짚어보기 좋다·정리하기 깔끔하다.

## 5. 문장 길이 변주

한 문단 안에 다음 길이를 의도적으로 섞는다.

- 짧은 문장 (15자 이하) — 호흡 끊고 강조
- 중간 문장 (30~50자) — 기본 흐름
- 긴 설명 문장 (80~150자) — 깊이 있는 부연

같은 길이의 문장을 3개 이상 연속으로 쓰지 않는다.
본문 중간에 짧은 질문형 ("왜 이런 차이가 생길까요?") 한두 번을 넣어 호흡을 만든다. 단, 도입부와 마무리에는 넣지 않는다.

## 6. 비유와 구체성

비유는 본문 전체에서 1회만 사용한다. 한 번 쓴 비유는 같은 글에서 다시 변주하지 않는다.
비유는 일상 사물에서 끌어온다 (책상, 신발, 컵, 옷, 가전). 추상 개념끼리 비유하지 않는다.

설명 안에 항상 구체 좌표를 넣는다.
- 시간 (오후 5시 / 한 시간 작업 후 / 일주일째)
- 거리 (1m / 모니터 거리 / 책상 위 서류)
- 신체 (코뼈 옆면 / 광대뼈 아래 / 귀 뒤 1cm)
- 환경 (형광등 아래 / 에어컨 바람 / 운전석 햇빛)

## 7. 가독성 10규칙

1. 한 문단은 3~4줄을 넘기지 않는다
2. 소제목은 명사형 또는 담백 서술형 (질문형 금지)
3. 강조(**볼드**)는 한 섹션에 1~2회 이내
4. 같은 문장 리듬 연속 반복 금지
5. 추상 개념은 예시 또는 비유로 즉시 구체화
6. 체크리스트는 표 또는 짧은 문단 형식 (불릿 폭주 금지)
7. 표는 본문 전체에서 1개, 3~4행
8. 표 앞뒤에서 표 내용을 다시 풀어 설명하지 않는다
9. 번호 매기기(1. 2. 3.)는 설명문에 사용하지 않는다 (체크리스트만 허용)
10. 쉼표는 꼭 필요할 때만, 접속사와 연결 어미로 문장을 잇는다

## 8. 마무리 — 정보형 안내

"안경사의 한마디" 라벨 한 줄 + 3~4문장:

1. 본문 핵심 압축 (1문장)
2. 개인차 또는 상황 차이 인정 (1문장)
3. 다음에 챙길 행동 또는 메모할 정보 (1~2문장)
4. (선택) 매장에서 확인하는 과정 한 줄

직접 전환 문구 금지: "상담", "문의", "예약", "지금 방문", "서두르세요".
정보형 대체: "현재 쓰는 안경을 함께 가져오면 비교가 편해요", "불편한 상황을 메모해두면 기준을 좁히기 좋아요".

매장 안내 블록은 본문 가장 마지막에 별도로 둔다. 안경사의 한마디와 내용이 겹치지 않게 한다.

## 9. 절대 금지

- 의료적 단정 ("치료된다", "완치된다", "100%")
- 비교·비방 ("다른 곳보다", "타 제품은")
- 자극 단어 ("충격", "필독", "꼭", "비용", "가격")
- 과장 강조 ("최고", "유일", "완벽", "절대")
- 학술 각주 ([1], URL 직접 삽입, 참고문헌 목록)
- 메타 안내문 ("YYYY-MM-DD 기준", "본 글은")
- AI 상투어 ("이번 글에서는", "차근차근", "정리해봤어요", "많은 분들이", "한 번쯤", "바쁜 일상")
- 작성 계획어 ("원인 후보", "판별 축", "확인 순서", "살펴봅니다")
