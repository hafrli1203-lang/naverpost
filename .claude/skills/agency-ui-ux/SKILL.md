# agency-ui-ux — 업무형 마케팅 대시보드 UI/UX 개선 스킬

> naverpost는 6개 안경원 블로그 콘텐츠를 만드는 **내부 운영자용 파이프라인 대시보드**다. 랜딩이 아니라 "입력→분석→결과→export" 흐름이 명확한 업무형 UI를 기준으로 본다.

## 목적
키워드→본문→이미지→export 3단계 파이프라인 화면을 **전문적·신뢰감 있게**, 단계 흐름이 명확하도록 개선한다.

## 디자인 기준
- 신뢰감 있는 절제된 업무 톤(과한 색/그라데이션/장식 금지) — AI Tells 회피.
- 한전ON형 서비스 포털 골격 + rightpeople형 행동 중심 진입.
- 기존 `src/components/ui/`(shadcn/ui, @base-ui/react) 토큰·컴포넌트 우선 재사용.

## 화면별 개선 기준
- 워크플로우 스테퍼: 현재/완료 단계 명확.
- 키워드 선택: 3개 옵션 비교·선택 쉽게.
- 본문 미리보기: 검증 결과(금지어/규칙 위반) 가시화 + 승인/수정 동선.
- 이미지 미리보기: 확인/재생성 동선.
- export: 복사/이미지 저장 + "네이버 임시저장" 안내(발행 아님 강조).

## 필수 점검
- 로딩(스켈레톤, 스피너 지양)/빈/오류 상태 3종 — AI·네이버 API·타임아웃 실패 포함.
- 반응형, 접근성(키보드/대비/ARIA), 다크모드(next-themes).
- 행동 중심 버튼명(키워드 생성/본문 작성/이미지 생성/export 복사).

## 절대 규칙
- 기존 기능/화면/API 라우트 삭제 금지. 네이버 발행 금지(export까지).
- 한 화면씩 분석→수정→검수(:3100 렌더)→기록(`docs/ai/DESIGN_AUDIT.md`).
- 테스트(렌더 확인) 없이 완료 보고 금지. 민감정보 열람·출력 금지.
